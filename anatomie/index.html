Vrstva 1 — Jak to funguje

pro všechny · bez žargonu

Model je jen textový prediktor

To, čemu říkáš „AI", je v jádru model — program, který dostane text a předpoví jeho pokračování. To je vše.

Nic „nedělá"
neotevře soubor, nepošle mail

Nemá paměť
nepamatuje si tě mezi zprávami

Nemá nástroje
text dovnitř → text ven

Žádný kouzelník — jen velmi dobrý „doplňovač" textu.

Harness — obal, který dá modelu ruce

To, co používáš (ChatGPT, Codex, Claude Code), je harness — program kolem modelu. Ten:

  • drží konverzaci
  • nabízí a spouští nástroje
  • vrací výsledky modelu
  • hlídá oprávnění

Model = mozek

Harness = tělo, ruce, smysly, sešit

Mozek sám nic neudělá — potřebuje tělo, které jedná.

Co model dostane na vstup — tři vrstvy

Než vyplodí jediné slovo, „přečte" tohle — v tomto pořadí:

1. System prompt — instrukce výrobce. Neviditelné, neměníš.

2. AGENTS.md / custom instructions — TVOJE vrstva. Vloží se na start každé session, ještě než AI uvidí tvou zprávu.

3. Tvoje zpráva — až teď přijde samotný dotaz.

Největší páka na lepší odpovědi = vrstva 2, ne chytřejší dotaz.

AGENTS.md — vrstva, kterou ovládáš

Napíšeš pravidla jednou do souboru → harness je automaticky vloží na start každé session. Nemusíš je opakovat v chatu.

Živý příklad: tahle „druhý mozek" složka má v kořeni AGENTS.md — proto se mi říká „Rod" a na startu načítám paměť.

Jiné jméno, stejný princip:

Codex → AGENTS.md
Claude Code → CLAUDE.md
ChatGPT → custom instructions

Otevřený standard (Linux Foundation, 60k+ projektů).

Kontext = pracovní paměť

Všechny tři vrstvy + historie + přiložené dokumenty + výstupy nástrojů žijí v jednom kontextu. Tři vlastnosti:

Omezený
má strop; co se nevejde, model nevidí

Ovlivňuje chování
balast = horší odpovědi

Stojí
posílá se znovu každý krok

Jak s kontextem pracovat (7 technik) → samostatný materiál „Práce s kontextem".

Session = znovuzrození

Každá session začíná od nuly. AI si tě ani minulý chat nepamatuje — jako by se pokaždé znovu narodila.

Jediná paměť = co znovu dostane do kontextu na startu. Proto AGENTS.md, paměťové soubory, projekty. A „čistý štít" je zároveň tajemství subagentů.

Turn = jedno tvé kolo, ale uvnitř smyčka

Není to „1 dotaz → 1 odpověď". Po tvé zprávě běží smyčka:


tvoje zpráva

model přemýšlí

harness spustí nástroj

výsledek zpět

↺ opakuje se, dokud není hotovo → teprve pak finální odpověď

Proto to „dlouho přemýšlí" (ne zásek) a vidíš ho číst soubory. Jeden turn = klidně desítky kroků.

Vrstva 2 — Mechanika hlouběji

pro znalostní pracovníky · práce v Codexu

Tool calling — jak model „jedná"

Model nikdy nic neudělá sám — jen požádá, harness vykoná.

MODEL
„spusť read_file(X)"
HARNESS
vykoná akci
výsledek
zpět do kontextu

Tohle je mechanika za Gen 3/4 z „Úrovní používání AI": model = mozek říká, harness = ruce dělají.

Subagenti a fork — mechanika

Subagent = samostatná session (vlastní kontext, vlastní smyčka). Do hlavní niti vrátí jen destilát. Nepořádek zůstane u něj.

fork  zdědí kontext rodiče.
fresh  startuje čistý + briefing.

Hlavní session (čistá)

↓ deleguje

Subagent — vlastní kontext + smyčka

„projdi 50 logů, vrať 5 bodů"

↑ vrátí jen destilát

Hlavní session pokračuje čistá

Volba: potřebuju, co už víme → fork. Chci nezaujatost / izolaci → fresh.

Proč se subagent vyplatí — 6 situací z praxe

Čistý kontext — špinavou práci udělá subagent, vrátí 5 bodů

Nezávislá kontrola — fresh subagent zreviduje tvou práci nezaujatě

Paralelně — N úkolů naráz, výsledky se slijí

Více úhlů — panel rolí (skeptik / bezpečnost / cena)

Riziko v izolaci — experiment ve vlastním prostoru

Nad rámec kontextu — audit stovek souborů

Headline: „druhý pár očí, co neseděl na schůzce" — fresh subagent nesdílí tvůj kontext, takže chytí slepá místa a confirmation bias.

Proč se občas zastaví a ptá na svolení

Některé akce (smazat soubory, push, citlivé operace) gatuje harness — ne model.

Je to bezpečnostní pojistka: o povolení rozhoduje obal, který má pravidla, ne samotný model. Proto tě to zastaví a čeká na tvé „ano".

Když chápeš, že rozhoduje harness (ne „rozmar AI"), víš i kde se to nastavuje.

Co si odnést

  • Největší páka = instrukce předem v AGENTS.md / custom instructions, ne doptávání za běhu.
  • Měníš téma → nová session. Čistý štít > tažení balastu.
  • „Dlouho přemýšlí" ≠ zásek — uvnitř běží smyčka.
  • Velký/špinavý úkol → deleguj. Chceš poctivou kontrolu → nech to zrevidovat fresh subagentem.
